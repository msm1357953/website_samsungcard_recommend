# 삼성카드 개인화 추천 웹사이트 기획안

## 프로젝트 개요

기존 삼성카드 "추천카드 TOP3" 사이트의 한계를 극복하고, **개인 맞춤형 카드 추천 서비스**를 구현합니다.

### 기존 사이트 문제점
| 문제 | 설명 |
|------|------|
| 제한된 추천 | 카테고리당 TOP3만 표시 (삼성카드 107종 중 극소수만 노출) |
| 단순한 혜택 표시 | 세분화된 혜택 정보 부족 |
| 정적 콘텐츠 | 사용자 니즈에 맞는 동적 필터링 불가 |

---

## 핵심 기능

### 1. 혜택 기반 동적 필터링
```
사용자가 원하는 혜택 선택 → 해당 혜택을 가진 카드 실시간 노출
```

**지원 혜택 카테고리 (9종):**
- 교통 / 쇼핑 / 스트리밍 / 영화 / 음식 / 주유 / 커피 / 통신 / 항공

### 2. 혜택 하이라이트 UI
- 혜택 버튼 클릭 시 → 해당 카드 목록 표시
- 카드별로 **해당 혜택 섹션만 강조 표시** (하이라이트)
- 할인율/할인금액 시각적 뱃지 처리

### 3. 카드 상세 정보 표시
- 카드 이미지 (고화질)
- 연회비 정보
- 전월 실적 조건
- 전체 혜택 목록 (선택형/기본 혜택 구분)

### 4. 카드 우선순위 (NEW)
네이버 카드검색 기준 **49개 우선순위 카드**를 상위 노출

| 구분 | 설명 |
|------|------|
| 우선순위 카드 (49개) | 네이버 광고 순위 기준 상위 노출 |
| 비우선순위 카드 (58개) | 우선순위 카드 다음에 표시 |
| 이름 매핑 | card-gorilla ↔ 네이버 이름 매핑 적용 |

**우선순위 적용 로직:**
```
1. 혜택 필터 → 해당 카드 필터링
2. 필터링된 카드 중 우선순위 카드 먼저 정렬
3. 우선순위 카드 내에서는 rank 순서
4. 비우선순위 카드는 연회비 낮은 순
```

## 검증 계획

### 기능 테스트
1. **필터링 테스트**: 각 혜택 버튼 클릭 시 정확한 카드만 표시되는지 확인
2. **하이라이트 테스트**: 선택한 혜택이 카드에서 강조 표시되는지 확인
3. **반응형 테스트**: 모바일/태블릿/데스크톱 화면 확인

### 수동 검증
- 브라우저에서 `index.html` 열기
- 각 혜택 필터 클릭하여 동작 확인
- 크롬 DevTools로 모바일 뷰 테스트
